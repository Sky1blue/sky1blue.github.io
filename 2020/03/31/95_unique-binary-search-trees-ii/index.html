95. Unique Binary Search Trees II

Given an integer n, generate all structurally unique BST's (binary search trees) that store values 1 ... n.

Example:


Input: 3

Output:

[

  [1,null,3,2],

  [3,2,null,1],

  [3,1,null,null,2],

  [2,1,3],

  [1,null,2,null,3]

]

Explanation:

The above output corresponds to the 5 unique BST's shown below:



   1         3     3      2      1

    \       /     /      / \      \

     3     2     1      1   3      2

    /     /       \                 \

   2     1         2                 3

思路

解法1：动态规划

分析：二叉搜索树的性质，左子树的所有值小于根节点，右子树的所有值大于根节点

分析来源

所以求 1…n 的所有可能。

我们只需要把 1 作为根节点，[ ] 空作为左子树，[ 2 … n ] 的所有可能作为右子树。

2 作为根节点，[ 1 ] 作为左子树，[ 3…n ] 的所有可能作为右子树。

3 作为根节点，[ 1 2 ] 的所有可能作为左子树，[ 4 … n ] 的所有可能作为右子树，然后左子树和右子树两两组合。

4 作为根节点，[ 1 2 3 ] 的所有可能作为左子树，[ 5 … n ] 的所有可能作为右子树，然后左子树和右子树两两组合。

…

n 作为根节点，[ 1… n ] 的所有可能作为左子树，[ ] 作为右子树。

至于，[ 2 … n ] 的所有可能以及 [ 4 … n ] 以及其他情况的所有可能，可以利用上边的方法，把每个数字作为根节点，然后把所有可能的左子树和右子树组合起来即可。

如果只有一个数字，那么所有可能就是一种情况，把该数字作为一棵树。而如果是 [ ]，那就返回 null。

解法2：动态规划

设V(n-1)为1…n-1得到的所有二叉搜索树，那么如何根据V(n-1)的结果得到V(n)呢。

由二叉搜索数的性质，直到根的值大于左子树和右子树的值，V(n)相较于V(n-1)增加了值n，应该是二叉树中最大的值，n必须放在数的最右边，

1) n作为root，将V(n-1)作为n的左子树即可；

2) n不是root，只要将n替代V(n-1)每层的右子树，同时将原来的右子树当作n的左子树。

例如 

n = 2,
output：

  [1,null,2]

  [2,1]

所以n=3,
3作为root，

  [3,1,null,null,2]

  [3,2,null,1]

3不作为root

  [1,null,2,null,3]

  [1,null,3,2],

  [2,1,3]
